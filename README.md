# 初心者にやさしく教えてくれる「メンターClaude」テンプレート

Claude Code / Claude（claude.ai）に、**プログラミング初心者を丁寧に伴走させる**ための
設定ファイル一式のテンプレートです。

「答え(完成コード)を一気に渡さず、1歩ずつ・行の意味を表で説明し・自分で打たせて・
構文チェックで確認しながら進める」——“家庭教師モード”を再現します


---

## 何が入っているか

| ファイル | 役割 | あなたが書き換える量 |
|---|---|---|
| `TEACHING.md` | **教え方・口調・進め方の型**（このテンプレの本体） | 少（技術例だけ差し替え） |
| `CLAUDE.md` | プロジェクトの説明・構成・環境の前提 | 中（`{{ }}` を自分の情報に） |
| `PROGRESS.md` | ロードマップと現在地（毎回更新する進捗表） | 多（自分の学習計画を書く） |

3ファイルは役割を分けてあります（1ファイルに全部書くと矛盾するため）。
- **教え方** は `TEACHING.md`（原則いじらない）
- **何を作っているか** は `CLAUDE.md`
- **今どこか** は `PROGRESS.md`（毎回更新）

---

## 使い方（3ステップ）

### 1. コピーする
`template/` の中の3ファイルを、あなたの学習プロジェクトの**ルート**に置きます。

```
your-project/
├── CLAUDE.md
├── TEACHING.md
└── PROGRESS.md
```

> Claude Code は、開いたリポジトリのルートにある `CLAUDE.md` を自動で読み込みます。
> `CLAUDE.md` の中から `@TEACHING.md` `@PROGRESS.md` を参照しているので、3つセットで効きます。

### 2. `{{ }}` を埋める
`CLAUDE.md` の中の `{{プロジェクト名}}` などの**プレースホルダ**を、自分の情報に置き換えます。
（`TEACHING.md` は基本そのままでOK。）
Claudeにプロジェクトを参照して CLAUDE.md　TEACHING.md　 PROGRESS.mdを更新してと言うほうが速いです。

### 3. 学習を始める
Claude に「続きから」と言えば、テンプレの型で伴走が始まります。

---

## 教え方をあなたの技術に合わせる（`TEACHING.md` の差し替え箇所）

`TEACHING.md` の教え方の“型”はどんな技術でも共通です。
ただし **例として出てくる技術名だけ** は React/JS 用のままなので、必要なら直してください。

- **構文チェックの手段**（`TEACHING.md` 内 `🔧 構文チェック` の項）
  例: Python なら `python -m py_compile file.py`、Go なら `go build` など、
  あなたの言語で「1コマンドで文法だけ確認できる方法」に置き換える。
- **「間違い方の傾向」セクション**
  これは “その人がよくやるミスを貯める場所” です。最初は空でよく、
  レビューで見つけたミスを1つずつ追記していくほど精度が上がります。
  （React/JS の例はサンプルとして残してあります。不要なら消してOK）

---

## この方針のポイント（なぜ効くのか）

- **答えを一気に渡さない**：完成コードを丸写しさせず、「今回の1歩」だけ見せて自分で打たせる。
  → 写経で終わらず、手が覚える。
- **行 | 意味 の対応表**：複数行のコードは必ず1行ずつ意味を説明する。
- **書くファイルのフルパスと行番号を毎回明示**：初心者が「どこに書くの?」で迷わない。
- **本人のミスの傾向を記録して使い回す**：同じ転び方を先回りで防げる。
